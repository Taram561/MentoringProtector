# Графический интерфейс: обзор

Приложение на Flutter/Dart (gui/) для Windows, без собственной логики обнаружения угроз - она вся в ядре, интерфейс отображает результаты и передаёт команды через FFI-мост.

## Слои


screens/    экраны и специфичные для экрана виджеты
providers/  состояние приложения на основе Provider
services/   бизнес-логика и обёртки над FFI
ffi/        мост к MentoringProtector_core.dll
models/     простые структуры данных (DTO)
widgets/    переиспользуемые визуальные компоненты
theme/      цвета, отступы, светлая/тёмная тема
l10n/       локализация (русский, английский)
utils/      мелкие вспомогательные функции


main.dart собирает приложение: провайдеры состояния, тема, локализация, корневой виджет.

Экраны обращаются не к FFI-мосту напрямую, а к сервисам, реализующим узкие интерфейсы (IScannerService, IQuarantineService и другие, см. [ffi.md](ffi.md)) - подмена реализации в тестах, независимость экранов от способа связи с ядром.

gui/windows/ - нативная "обвязка" Flutter для Windows и точка входа процесса (windows/runner/main.cpp); вся логика приложения - в Dart-коде.
