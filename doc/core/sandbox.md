# Песочница

## Файлы

| Файл | Описание |
|---|---|
| sandbox_manager.cpp / sandbox_manager.h | Координация запуска, отслеживание состояния, формирование итогового отчёта |
| sandbox_process.cpp / sandbox_process.h | Запуск изолированного процесса: Job Object, AppContainer, определение типа файла |
| sandbox_monitor.cpp / sandbox_monitor.h | Наблюдение за поведением запущенного процесса, вычисление оценки риска |

## Классы и функции

### SandboxManager (sandbox_manager.h / sandbox_manager.cpp)

| Метод                                                                | Описание                                                                           |
| -------------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| instance() (статический)                                             | Единственный экземпляр                                                             |
| ~SandboxManager()                                                    | Отмена запуска, если ещё выполняется, ожидание потока-наблюдателя                  |
| isSupported() const                                                  | Проверка доступности CreateAppContainerProfile в userenv.dll                       |
| run(filePath, timeoutSeconds)                                        | См. разбор ниже                                                                    |
| cancel()                                                             | Остановка раньше таймаута, самостоятельное формирование отчёта (timed_out = false) |
| getState() const / getElapsedSeconds() const / getReportJson() const | Состояние/время работы/отчёт под мьютексом                                         |
| escapeJson(s) const                                                  | Экранирование строки для JSON вручную                                              |

**run(filePath, timeoutSeconds)** - при уже идущем запуске - ошибка already_running; создание SandboxProcess, запуск с лимитом памяти 256 МБ, создание SandboxMonitor, запуск watcherLoop.

**watcherLoop(timeoutSecs)** - раз в секунду: проверка отмены, истечения таймаута, жив ли процесс; по завершении - принудительное завершение процесса, остановка монитора, формирование отчёта.

**buildReportJson(...)** - помимо сериализации событий строит человекочитаемый список индикаторов риска (число дочерних процессов, загруженных модулей, всплеск памяти) в готовые фразы.

### SandboxProcess (sandbox_process.h / sandbox_process.cpp)

| Метод                                      | Описание                                                                                                   |
| ------------------------------------------ | ---------------------------------------------------------------------------------------------------------- |
| SandboxProcess() / ~SandboxProcess()       | Деструктор завершает процесс, закрывает дескрипторы, уничтожает AppContainer, удаляет временный каталог    |
| launch(cfg)                                | См. разбор ниже                                                                                            |
| terminate()                                | TerminateProcess + ожидание до 3 секунд                                                                    |
| isRunning() const                          | Проверка кода завершения (GetExitCodeProcess)                                                              |
| getPid() const                             | PID запущенного процесса                                                                                   |
| toLowerExt(path) (статический)             | Расширение файла в нижнем регистре                                                                         |
| winErrorToCode(err) (статический)          | Код ошибки Windows в короткий текстовый код                                                                |
| setupJobObject(memMB)                      | Job Object с лимитами: завершение всех процессов при закрытии, лимит памяти, не более 5 активных процессов |
| createTempDir()                            | Уникальный рабочий каталог в %TEMP%\MPSandbox\<GUID>\                                                      |
| deleteDir(dir)                             | Рекурсивное удаление со снятием атрибута "только для чтения"                                               |
| destroyAppContainer()                      | Освобождение SID, удаление профиля AppContainer                                                            |
| grantWorkDirAccessToAppContainer(dir, sid) | Добавление в ACL рабочего каталога прав на чтение+выполнение для SID контейнера                            |
| close()                                    | Закрытие дескрипторов процесса, потока, задания                                                            |

**launch(cfg)** - определение типа файла (DLL отклоняется) -> копирование во временный рабочий каталог -> создание AppContainer с доступом только к рабочему каталогу -> Job Object с лимитом памяти -> CreateProcessW с CREATE_SUSPENDED и привязкой к AppContainer -> AssignProcessToJobObject -> ResumeThread; порядок гарантирует, что процесс не успевает действовать до наложения всех ограничений.

**detectFileKind(path)** - для PE-файлов разбор заголовка (флаг IMAGE_FILE_DLL) отличает исполняемый файл от DLL независимо от расширения.

**createAppContainer(outName, outSid)** - уникальное имя профиля на основе GUID; при уже существующем профиле - получение SID вместо ошибки.

### SandboxMonitor (sandbox_monitor.h / sandbox_monitor.cpp)

| Метод                                    | Описание                                                               |
| ---------------------------------------- | ---------------------------------------------------------------------- |
| SandboxMonitor(sandbox_pid)              | Сохранение PID для наблюдения                                          |
| ~SandboxMonitor()                        | Вызов stop()                                                           |
| start()                                  | Запуск фонового потока опроса                                          |
| stop()                                   | Остановка потока с ожиданием                                           |
| getEvents() const                        | Копия накопленных событий                                              |
| pollLoop()                               | Раз в секунду - все три проверки ниже                                  |
| checkModules()                           | Новые модули вне системных каталогов - как события (лимит 512 модулей) |
| checkMemory()                            | Превышение рабочего набора 200 МБ - однократное событие                |
| addEvent(type, target, detail)           | Добавление события с меткой времени                                    |
| timestamp() const / wstrToUtf8(ws) const | Форматирование времени, преобразование строк                           |
| isSystemPath(lower) const                | Путь модуля в системном каталоге                                       |

**checkChildProcesses()** - рекурсивный обход всего дерева потомков, а не только прямых детей.

**computeRiskScore() const** - дочерние процессы (20 за штуку, потолок 60), нестандартные модули (5 за штуку, потолок 45), всплеск памяти (фиксированные 25); итог ограничен 100.
